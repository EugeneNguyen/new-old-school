# REQ-009: Stream Reconnect on Session Reload

## Description
When a user navigates away from an active streaming session and comes back, the client should reconnect to the live stream and continue showing real-time output.

## Acceptance Criteria
- [x] Server tracks active Claude subprocesses in an in-memory registry
- [x] A status endpoint reports whether a session is currently streaming
- [x] A reconnect SSE endpoint replays missed events from disk then pipes live output
- [x] Client checks stream status when loading a session and reconnects if active
- [x] Client cleans up fetch on component unmount
- [x] Completed sessions load static history as before

## Technical Constraints
- Registry is in-memory (ephemeral) u2014 server restart clears it
- Must not break existing streaming flow for the original client
- SSE format must match existing event format for client reuse
