# REQ-010: Interactive Question UI in Terminal

## Description
Add interactive question handling to the Claude Terminal so that when Claude Code emits
`AskUserQuestion` tool_use events in the SSE stream, the terminal renders clickable UI
elements (buttons for single-select, checkboxes for multi-select) and sends the user's
response as a follow-up message in the same session.

Note: `--dangerously-skip-permissions` remains enabled. The CLI auto-denies AskUserQuestion,
but the questions are detected and rendered. User answers are sent as follow-up messages.

## Acceptance Criteria
- [x] Terminal detects `AskUserQuestion` tool_use events in SSE stream
- [x] Renders single-select question UI with clickable option buttons
- [x] Renders multi-select question UI with checkboxes and submit button
- [x] Renders freeform text input when no options provided
- [x] User selection sent as follow-up message in same session
- [x] Regular message streaming resumes after user responds
- [x] Answered state shown with green checkmark and selected values

## Technical Constraints
- Works within existing SSE streaming architecture (no backend changes)
- Uses shadcn/ui + Tailwind for UI components
- TypeScript with @/* path aliases
