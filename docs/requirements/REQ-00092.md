# REQ-00092: File System Browser UI

## Description

Enhance the workspace file browser to allow users to browse both files and folders inside the workspace. Currently, the `FolderBrowser` component only shows subdirectories; this extension adds file display, file type icons, and navigation capabilities.

## User Stories

### As a user, I want to browse files in my workspace

**Scenario:** I am creating a new workspace and want to select an existing directory
**Given:** I am on the Workspaces page
**When:** I click on "New Workspace"
**Then:** I see a file browser that shows both folders and files

### As a user, I want to preview file contents

**Scenario:** I want to check the contents of a file before selecting a workspace
**Given:** I am browsing a directory with files
**When:** I click on a file
**Then:** I see a preview panel showing the first 100 lines of the file

### As a user, I want to search for specific files

**Scenario:** I have a large directory and need to find a specific file
**Given:** I am browsing a directory
**When:** I type in the search box
**Then:** Only files matching my search query are displayed

## Acceptance Criteria

- [x] Files are displayed alongside folders in the browser
- [x] File icons reflect file type based on extension
- [x] File size is displayed and formatted appropriately
- [x] Folders and files are sorted: folders first, then alphabetically
- [x] Clicking a file shows a preview panel with file content
- [x] Clicking a folder navigates into it
- [x] Up button navigates to parent directory
- [x] Refresh button reloads current directory
- [x] Search input filters displayed entries in real-time
- [x] Preview panel can be closed
- [x] Follows UI design standards (colors, spacing, typography from `docs/standards/ui-design.md`)

## Technical Constraints

- Next.js App Router + TypeScript with `@/*` path aliases
- shadcn/ui + Tailwind; Lucide icons
- Existing `/api/workspaces/browse` endpoint (no backend changes needed)
- Integration with `components/ui/scroll-area.tsx` for overflow containment
- Mobile-friendly: preview panel below file list on small screens

## Implementation Details

### API Changes

The browse endpoint now returns entries with additional metadata:

```typescript
interface BrowseEntry {
  name: string;
  absolutePath: string;
  isDirectory: boolean;
  size?: number;
  modified?: string;
}
```

### File Icons

Files are displayed with appropriate icons based on extension:
- `FileText` (blue): markdown, txt, rst, doc, docx
- `FileCode` (green): js, jsx, ts, tsx, json, yaml, yml, toml, css, html, xml
- `FileImage` (purple): png, jpg, jpeg, gif, svg, webp, ico
- `File` (gray): all other file types

### Preview Panel

The preview panel shows:
- File name and path
- File size (formatted)
- Last modified date (relative format)
- First 100 lines of text files
- "Preview not available" message for binary files
