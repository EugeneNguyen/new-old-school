# REQ-012: Workflow Foundation

## Description
Establish a structured system for managing "workflows" within the project, starting with a "Requirements" workflow. This system will track items as they move through configurable stages, each with its own internal status (Todo, In Progress, Done).

## Acceptance Criteria
- [ ] A `.nos/` directory exists at the project root.
- [ ] A `.nos/workflows/` directory exists.
- [ ] A `.nos/workflows/requirements/` directory exists.
- [ ] A configuration mechanism (file-based) exists to define stages for a workflow.
- [ ] A data structure/file system exists to track items within a workflow.
- [ ] Items support:
    - Association with a specific stage.
    - A status within that stage (`Todo`, `In Progress`, `Done`).
    - A way to store comments.
- [ ] Logic (or a defined rule) is established such that moving an item to a new stage resets its status to `Todo`.

## Technical Constraints
- The system should be "file-first," meaning the state is persisted in the filesystem within the `.nos` directory.
- Should be designed for extensibility (other workflows can be added similarly to `requirements`).
