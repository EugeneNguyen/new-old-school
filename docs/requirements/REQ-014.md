# REQ-014: Workflow Item Detail Modal & Add Item

## Description
On the workflow Kanban screen, clicking an item card opens a detail modal/popup showing the item's title, description (markdown body of `index.md`), and comments, with a side panel listing the workflow's stages (current one highlighted) and status options. The modal supports editing these fields and persisting changes via a Save button, plus a Close button to dismiss. The Kanban screen also gains an "Add item" action that creates a new item in the first stage.

## Acceptance Criteria
- [x] Clicking an item card on `/dashboard/workflows/[id]` opens a modal dialog.
- [x] Modal displays the item's title (editable) and description from `index.md` (editable as plain markdown in a textarea).
- [x] Modal shows the list of existing comments and allows adding a new comment.
- [x] Side panel shows all workflow stages in order with the current stage highlighted; clicking a stage reassigns the item to it.
- [x] Side panel shows status options (`Todo` / `In Progress` / `Done`) and allows changing the status.
- [x] A Save button persists all edits: title/stage/status/comments via PATCH meta, body via PUT content.
- [x] A Close button (and backdrop/Esc) closes the modal; after Save, the underlying Kanban reflects the changes.
- [x] An "Add item" button on the Kanban page opens a simple form (title required) that creates a new item in the first stage with `Todo` status and empty body/comments.
- [x] Newly created items appear immediately in the first stage column without a full page reload.
- [x] Unknown workflow id / missing items surface errors rather than crashing.

## Technical Constraints
- Next.js App Router; modal implemented as a client component under `components/dashboard/`.
- New lightweight `Dialog` primitive in `components/ui/dialog.tsx` (no new heavy dependency; native fixed overlay + Esc/backdrop close).
- Reuse existing API:
  - `PATCH /api/workflows/[id]/items/[itemId]` — update title/stage/status/comments.
  - `PUT /api/workflows/[id]/items/[itemId]/content` — update body.
- New API:
  - `POST /api/workflows/[id]/items` — body `{ title, id? }`; creates folder `.nos/workflows/[id]/items/[itemId]/` with `meta.yml` (stage = first stage, status = Todo, empty comments) and empty `index.md`. If `id` omitted, slugify `title`; append `-N` on collision.
- Extend `lib/workflow-store.ts` with `createItem(workflowId, { title, id? })`.
- Follow REQ-012 file-first persistence and REQ-013 conventions.
- Use existing shadcn primitives (Button, Badge, Input, Card).
