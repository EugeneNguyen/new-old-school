# REQ-006: Terminal Session Management

## Description
Add session management capabilities to the Claude Terminal screen. Users need to view all sessions observed by the system, create new sessions, and resume/switch to any previous session.

## Acceptance Criteria
- [x] API endpoint (`GET /api/claude/sessions`) lists all sessions from `.claude/sessions/` with metadata (ID, created date, modified date, message preview)
- [x] Session sidebar/panel in the Terminal page displays all available sessions
- [x] Users can create a new session (clears current chat, starts fresh)
- [x] Users can click an existing session to resume it (loads history + sets sessionId for `--resume`)
- [x] Currently active session is visually highlighted in the session list
- [x] Session list refreshes after each new conversation starts

## Technical Constraints
- Next.js App Router + TypeScript with `@/*` path aliases
- shadcn/ui + Tailwind CSS for UI components
- Sessions stored in `.claude/sessions/` as `.txt` files (stream-json format)
- Must use existing `--resume {sessionId}` CLI flag for conversation continuity
- Parse session `.txt` files to extract displayable message history
