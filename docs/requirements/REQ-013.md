# REQ-013: Workflow Kanban Screen

## Description
When a user clicks a workflow in the sidebar's Workflows section, they are taken to a dedicated workflow screen that renders a Kanban board. Each column represents a stage (from the workflow's `stages.yaml`), and each item in the workflow is rendered as a card inside its current stage's column. Items can be dragged between stages; the change is persisted to the filesystem.

## Acceptance Criteria
- [x] Route `/dashboard/workflows/[id]` renders for any workflow listed by `/api/workflows`.
- [x] Page reads stages (ordered) from `.nos/workflows/[id]/config/stages.yaml`.
- [x] Page reads items from `.nos/workflows/[id]/items/[itemId]/` folders (`meta.yml` for `title`/`stage`/`status`/`comments`, `index.md` for content).
- [x] Each stage renders as a Kanban column in the stages.yaml order, with a header showing the stage name and item count.
- [x] Each item renders as a card inside its stage's column, showing title and status badge (`Todo` / `In Progress` / `Done`).
- [x] Items are draggable between columns. Dropping onto a new column:
  - [x] Updates the item's `stage` in its `meta.yml`.
  - [x] Resets the item's `status` to `Todo` (per REQ-012 rule).
  - [x] Persists the change via a PATCH endpoint at `/api/workflows/[id]/items/[itemId]`.
- [x] Unknown workflow id renders a "not found" message, not a 500.
- [x] Empty stages render an empty-state placeholder.
- [x] Existing sidebar link (`/dashboard/workflows/[id]`) navigates to this screen.

## Technical Constraints
- Next.js App Router dynamic route (`app/dashboard/workflows/[id]/page.tsx`).
- Server-side data reads (`fs` + `js-yaml`) for stages; item list fetched via API for client-side state after drag.
- New client component for the Kanban board — uses native HTML5 drag-and-drop (no new heavy dep) with optimistic update + rollback on error.
- New API routes:
  - `GET /api/workflows/[id]` — returns `{ id, name, stages, items }`.
  - `PATCH /api/workflows/[id]/items/[itemId]` — body `{ stage }`; updates frontmatter and resets status.
- Reuses shadcn `Card` and `Badge` components.
- Add `js-yaml` dependency if not already present.
- Follow REQ-012: file-first state in `.nos/workflows/`.
