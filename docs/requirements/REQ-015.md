# REQ-015: Editable Stage Information from Kanban Column

## Description
On the workflow Kanban screen, each stage column header has a three-dot (⋯) icon button. Clicking it opens a modal/popup that displays the stage's information (name, description, prompt, autoAdvanceOnComplete) in editable form and persists changes to `.nos/workflows/[id]/config/stages.yaml`. After saving, the Kanban board reflects the updated stage without a full page reload, and any items that were on a renamed stage stay attached to the renamed stage.

## Acceptance Criteria
- [x] Each Kanban stage column header renders a three-dot icon button next to the item count.
- [x] Clicking the three-dot icon opens a modal dialog scoped to that stage.
- [x] Modal shows editable fields: `name` (text), `description` (textarea), `prompt` (textarea), `autoAdvanceOnComplete` (checkbox).
- [x] Save button persists changes via `PATCH /api/workflows/[id]/stages/[stageName]`.
- [x] On save, the stage is updated in `stages.yaml` preserving stage order.
- [x] If the stage name changes, every item whose `stage` matches the old name is updated to the new name in its `meta.yml`.
- [x] Kanban board reflects updated stage name/description/item grouping after save without a full page reload.
- [x] Close button, backdrop click, and Esc key all dismiss the modal.
- [x] Clicking the three-dot icon does not trigger the column's drag/drop handlers or open the item detail modal.
- [x] Invalid requests (unknown workflow, unknown stage, empty name, duplicate name) return a 4xx error and the modal surfaces the message.

## Technical Constraints
- New client component `components/dashboard/StageDetailDialog.tsx` using the existing `Dialog`, `Button`, `Input` primitives.
- Three-dot icon uses Lucide `MoreVertical`.
- New API route `app/api/workflows/[id]/stages/[stageName]/route.ts` with a `PATCH` handler.
- Extend `lib/workflow-store.ts` with `updateStage(workflowId, stageName, patch)` that:
  - Rewrites `config/stages.yaml` preserving array order.
  - Rewrites `items/*/meta.yml` when the stage is renamed so items stay attached.
  - Returns the updated `Stage[]` (and optionally updated items).
- `KanbanBoard` holds `stages` in local state so updates propagate without a reload; parent page continues to SSR the initial list.
- Follows REQ-012 file-first persistence and REQ-013 Kanban conventions.
